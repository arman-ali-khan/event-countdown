📋 Overview

Event Countdown Builder is a comprehensive, modern web application that allows users to create stunning countdown pages for any special event. Built with the latest web technologies including React 18, TypeScript, Vite, and TailwindCSS, it offers a seamless experience for both creators and visitors with advanced features like user management, admin dashboard, and event analytics.

🎨 Customization Guide

Colors and Themes

The application uses a comprehensive color system. You can customize event type colors by modifying the getThemeColors() function in src/components/CountdownTimer.tsx:

const getThemeColors = () => { switch (eventType) { case 'wedding': return 'from-rose-500 to-pink-600'; // Wedding theme case 'birthday': return 'from-purple-500 to-indigo-600'; // Birthday theme case 'product-launch': return 'from-green-500 to-emerald-600'; // Product launch theme default: return 'from-blue-500 to-cyan-600'; // Custom theme } };

Typography

Update fonts in tailwind.config.js:

fontFamily: { sans: ['Your-Font', 'system-ui', 'sans-serif'], mono: ['Your-Mono-Font', 'monospace'], },

Adding New Event Types (Admin Panel)

Use the admin panel to create custom event types:

  1. Login as admin and go to Settings
  2. Navigate to "Event Type Management"
  3. Click "Create Type" button
  4. Fill in the details (value, label, icon, color)
  5. Save settings to apply changes

Adding New Event Types (Code)

Alternatively, modify src/components/EventForm.tsx:

const eventTypes = [ { value: 'wedding', label: 'Wedding', icon: Heart, color: 'text-rose-500' }, { value: 'birthday', label: 'Birthday', icon: Gift, color: 'text-purple-500' }, { value: 'product-launch', label: 'Product Launch', icon: Rocket, color: 'text-green-500' }, { value: 'conference', label: 'Conference', icon: Users, color: 'text-blue-500' }, // New type ];

Custom Animations

Customize animations in tailwind.config.js:

animation: { 'custom-bounce': 'customBounce 2s infinite', 'custom-fade': 'customFade 0.5s ease-in-out', }, keyframes: { customBounce: { '0%, 100%': { transform: 'translateY(0px)' }, '50%': { transform: 'translateY(-10px)' }, }, },
Pro Tip: Use the admin panel's settings section to customize event types, system limits, and other configurations without touching the code.

🔌 API Integration

Current Data Storage

The application currently uses localStorage for data persistence, which is perfect for demos and small-scale deployments. For production use, you can integrate with various backend services:

Recommended Backend Services

  • Supabase: PostgreSQL database with real-time features
  • Firebase: Google's NoSQL database with authentication
  • MongoDB Atlas: Cloud MongoDB with REST API
  • Strapi: Headless CMS with customizable API
  • Custom REST API: Node.js/Express or Python/Django backend

Integration Points

Key areas that would benefit from backend integration:

  • User Authentication: Replace localStorage with JWT tokens
  • Event Storage: Store events in a database with proper relationships
  • Image Upload: Use cloud storage (AWS S3, Cloudinary)
  • Email Services: Integrate with SendGrid, Mailgun, or AWS SES
  • Analytics: Add Google Analytics or custom tracking
  • Real-time Updates: WebSocket connections for live countdown updates

Example API Structure

// Example API endpoints GET /api/events // Get all public events POST /api/events // Create new event GET /api/events/:id // Get specific event PUT /api/events/:id // Update event DELETE /api/events/:id // Delete event GET /api/users/me // Get current user POST /api/auth/login // User login POST /api/auth/register // User registration POST /api/auth/logout // User logout GET /api/events/:id/joins // Get event join requests POST /api/events/:id/join // Join an event

🚀 Deployment Guide

Vercel Deployment (Recommended)

  1. Install Vercel CLI: npm i -g vercel
  2. Run: vercel --prod
  3. Configure:
    • Build Command: npm run build
    • Output Directory: dist
    • Install Command: npm install

Netlify Deployment

  1. Build the project: npm run build
  2. Deploy via Netlify CLI:
    npm install -g netlify-cli netlify deploy --prod --dir=dist
  3. Or drag and drop the dist folder to Netlify

Static Hosting (Apache/Nginx)

  1. Build the project: npm run build
  2. Upload dist folder contents to your web server
  3. Configure routing for SPA support:
    # Apache (.htaccess) Options -MultiViews RewriteEngine On RewriteCond %{REQUEST_FILENAME} !-f RewriteRule ^ index.html [QSA,L] # Nginx location / { try_files $uri $uri/ /index.html; }

Environment Variables

For production deployment, you may want to set up environment variables:

# .env.production VITE_API_URL=https://your-api-url.com VITE_ANALYTICS_ID=your-analytics-id VITE_SUPABASE_URL=your-supabase-url VITE_SUPABASE_ANON_KEY=your-supabase-key
Note: The application uses localStorage for data persistence. For production use, consider integrating with a backend database service.

🔧 Troubleshooting

Common Issues

Build fails with memory error

NODE_OPTIONS="--max-old-space-size=4096" npm run build

Dark mode not working

  • Ensure dark class is added to HTML element
  • Check localStorage for theme preference
  • Verify TailwindCSS dark mode configuration

Countdown not updating

  • Verify date format is correct (ISO 8601)
  • Check browser console for JavaScript errors
  • Ensure device date/time is correct

Images not uploading

  • Check file size (default limit: 10MB)
  • Verify file format (PNG, JPG supported)
  • Check browser console for errors

Social sharing not working

  • Ensure HTTPS in production
  • Check popup blocker settings
  • Verify social media URLs are correct

Admin panel not accessible

  • Verify admin credentials (admin@countdownbuilder.com / admin123)
  • Check if user has admin privileges in localStorage
  • Clear browser cache and try again

Performance Optimization

  • Images are lazy-loaded where possible
  • Components are optimized to prevent unnecessary re-renders
  • Animations use CSS transforms for better performance
  • Code splitting is implemented for optimal loading
  • Admin components are loaded only when needed
Need Help? If you encounter issues not covered here, please check the support section or contact us with detailed error information.

🚀 Installation Guide

Step 1: Download and Extract

  1. Download the theme package from ThemeForest
  2. Extract the ZIP file to your desired location

Step 2: Install Dependencies

🎨 Customization Guide

Colors and Themes

The application uses a comprehensive color system. You can customize event type colors by modifying the getThemeColors() function in src/components/CountdownTimer.tsx:

const getThemeColors = () => { switch (eventType) { case 'wedding': return 'from-rose-500 to-pink-600'; // Wedding theme case 'birthday': return 'from-purple-500 to-indigo-600'; // Birthday theme case 'product-launch': return 'from-green-500 to-emerald-600'; // Product launch theme default: return 'from-blue-500 to-cyan-600'; // Custom theme } };

Typography

Update fonts in tailwind.config.js:

fontFamily: { sans: ['Your-Font', 'system-ui', 'sans-serif'], mono: ['Your-Mono-Font', 'monospace'], },

Adding New Event Types

Add new event types through the admin panel or by modifying src/components/EventForm.tsx:

const eventTypes = [ { value: 'wedding', label: 'Wedding', icon: Heart, color: 'text-rose-500' }, { value: 'birthday', label: 'Birthday', icon: Gift, color: 'text-purple-500' }, { value: 'product-launch', label: 'Product Launch', icon: Rocket, color: 'text-green-500' }, { value: 'conference', label: 'Conference', icon: Users, color: 'text-blue-500' }, // New type ];

Custom Animations

Customize animations in tailwind.config.js:

animation: { 'custom-bounce': 'customBounce 2s infinite', 'custom-fade': 'customFade 0.5s ease-in-out', }, keyframes: { customBounce: { '0%, 100%': { transform: 'translateY(0px)' }, '50%': { transform: 'translateY(-10px)' }, }, },
Pro Tip: Use the admin panel's settings section to customize event types, system limits, and other configurations without touching the code.

🚀 Deployment Guide

Vercel Deployment (Recommended)

  1. Install Vercel CLI: npm i -g vercel
  2. Run: vercel --prod
  3. Configure:
    • Build Command: npm run build
    • Output Directory: dist
    • Install Command: npm install

Netlify Deployment

  1. Build the project: npm run build
  2. Deploy via Netlify CLI:
    npm install -g netlify-cli netlify deploy --prod --dir=dist
  3. Or drag and drop the dist folder to Netlify

Static Hosting (Apache/Nginx)

  1. Build the project: npm run build
  2. Upload dist folder contents to your web server
  3. Configure routing for SPA support:
    # Apache (.htaccess) Options -MultiViews RewriteEngine On RewriteCond %{REQUEST_FILENAME} !-f RewriteRule ^ index.html [QSA,L] # Nginx location / { try_files $uri $uri/ /index.html; }

Environment Variables

For production deployment, you may want to set up environment variables:

# .env.production VITE_API_URL=https://your-api-url.com VITE_ANALYTICS_ID=your-analytics-id
Note: The application uses localStorage for data persistence. For production use, consider integrating with a backend database service.

🔧 Troubleshooting

Common Issues

Build fails with memory error

NODE_OPTIONS="--max-old-space-size=4096" npm run build

Dark mode not working

  • Ensure dark class is added to HTML element
  • Check localStorage for theme preference
  • Verify TailwindCSS dark mode configuration

Countdown not updating

  • Verify date format is correct (ISO 8601)
  • Check browser console for JavaScript errors
  • Ensure device date/time is correct

Images not uploading

  • Check file size (default limit: 10MB)
  • Verify file format (PNG, JPG supported)
  • Check browser console for errors

Social sharing not working

  • Ensure HTTPS in production
  • Check popup blocker settings
  • Verify social media URLs are correct

Performance Optimization

  • Images are lazy-loaded where possible
  • Components are optimized to prevent unnecessary re-renders
  • Animations use CSS transforms for better performance
  • Code splitting is implemented for optimal loading
